# User Preference

用于记录用户在项目协作中的固定偏好，后续新对话开始时优先读取。

## 固定偏好
1. 用户习惯将已知 BUG 记录在 `BUG.md` 中。
2. 用户习惯在模型对代码做修改前，先在 `plan.md` 里做明确规划，再在 `todo.md` 里生成详细可实现的 ToDo List，随后按 ToDo List 一步步实施代码修改。
3. BUG.md、plan.md中生成的内容都用中文，todo.md无硬性要求
4. 注释尽可能用中文，commit也是。
## 维护说明
- 新偏好统一追加在本文件中，避免散落在聊天记录里。
- 如果偏好变化，保留旧项并标注“已废弃”和生效日期。
